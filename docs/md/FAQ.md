# FAQ

## What is a Code App?

A new Power Platform app type. You write a real SPA (React, Vue, etc.) in your IDE and deploy it to Power Platform. It runs in a sandboxed iframe with Entra ID auth handled by the host. Requires Power Apps Premium license.

## How is this different from canvas/model-driven apps?

Canvas and model-driven apps use low-code designers. Code Apps are full pro-dev — you write TypeScript, use npm packages, and deploy via CLI. The platform provides auth, governance (DLP, Conditional Access), and data connectivity.

## Why can't I use `fetch()` to call Dataverse?

The app runs in a sandboxed iframe — there are no auth tokens available to your code. All Dataverse calls must go through the generated `AccountsService` (or equivalent), which delegates to the Power Apps host for authentication.

## What's the difference between database references and connector references?

- **Database references** — for Dataverse. First-class path, no connection IDs needed. Use `pac code add-data-source -a dataverse -t <table>`.
- **Connector references** — for everything else (SQL, SharePoint, O365). Requires pre-creating connections in make.powerapps.com.

Do NOT use `shared_commondataserviceforapps` connector for Dataverse — it compiles but silently fails at runtime.

## Why not use `$select`?

Including computed fields (formatted values, `_value` lookups) in `$select` causes Dataverse to return zero rows with no error. Omitting `$select` returns all columns — safe for a demo with the account table.

## How do I run locally?

Two terminals: `npm run dev` (Vite on port 3000) + `pac code run` (Power Platform proxy). Open the URL from `pac code run`, not the Vite URL.

## How do I deploy?

```bash
npm run build
pac code push
```

## Can I use `npx power-apps` instead of `pac code`?

No. The `npx power-apps` npm CLI does not exist despite some documentation references. Use `pac code` for all operations (init, push, run, add-data-source).

## What's in the `.power/` directory?

Auto-generated schemas used by the SDK at runtime. Don't edit — they're regenerated by `pac code add-data-source`.

## What's in `src/generated/`?

Auto-generated TypeScript services and models for each registered Dataverse table. Don't edit — they're regenerated by `pac code add-data-source`. The service provides `create`, `get`, `getAll`, `update`, `delete`, and `getMetadata` methods.

## How do dashboard tooltips and drilldown cards work?

Hover any dashboard tile to see a tooltip previewing the underlying data. Click to open a drilldown dialog with a full filtered table. Tooltips use pure CSS (`group-hover/tip`), not a library. KPI cards use `position="below"` to avoid viewport clipping; chart sub-elements use `position="above"`.

## How does the AI "Extract Action Items" feature work?

Open a meeting summary detail view and click the purple **Extract Action Items** button. The app sends the meeting notes to Azure OpenAI, which returns structured action items (name, priority, due date, notes). You preview them in a table with checkboxes, deselect any you don't want, and click **Create**. Records are written to the `tdvsp_actionitem` table linked to the same account. Requires `VITE_AOAI_*` env vars — see `.env.example`.

## How does the command palette work?

Press **Ctrl+K** (or **Cmd+K** on Mac) anywhere in the app. A search modal opens with fuzzy search across all five entity types. Results are grouped by entity, with icons and subtitles. Arrow keys to navigate, Enter to select, Escape to close. It searches TanStack Query's cached data — no extra API calls.

## My deployed app shows a blank white screen but works locally?

Vite defaults to `base: "/"`, which generates absolute asset paths (`/assets/index.js`). Power Platform serves Code Apps from a nested hosting path, so those paths 404. Set `base: "./"` in `vite.config.ts` to use relative paths. Local dev isn't affected because `pac code run` proxies to `localhost:3000` where root paths resolve naturally.
