# Architecture

## Overview

This is a Power Platform Code App — a React SPA that runs inside an iframe hosted by Power Apps. The platform handles Entra ID authentication; the app communicates with Dataverse through generated service classes provided by the `@microsoft/power-apps` SDK.

## Runtime Model

```
┌─────────────────────────────────────┐
│       Power Apps Host (iframe)      │
│                                     │
│  ┌───────────────────────────────┐  │
│  │     React SPA (this app)     │  │
│  │                               │  │
│  │  AccountList ──► useAccounts  │  │
│  │                    │          │  │
│  │            AccountsService    │  │
│  │             (generated)       │  │
│  │                    │          │  │
│  │          @microsoft/power-apps│  │
│  │              SDK client       │  │
│  └──────────────┬────────────────┘  │
│                 │                    │
│        Host-managed auth            │
│         (Entra ID tokens)           │
│                 │                    │
│           Dataverse API             │
└─────────────────────────────────────┘
```

## Data Flow

1. React components call TanStack Query hooks (`useAccounts`, `useCreateAccount`, etc.)
2. Hooks call static methods on `AccountsService` (auto-generated)
3. `AccountsService` uses `getClient()` from `@microsoft/power-apps/data` to make Dataverse API calls
4. The SDK client delegates to the Power Apps host for authentication — no token management in app code
5. TanStack Query handles caching, deduplication, and invalidation

## AI Integration (Azure OpenAI)

```
Meeting Summary Detail → "Extract Action Items" button
        │
  src/lib/azure-openai.ts
        │
  Azure OpenAI (Chat Completions API)
        │
  JSON response → ExtractedActionItem[]
        │
  Preview dialog (checkboxes) → Bulk create
        │
  Tdvsp_actionitemsService.create() × N
        │
  TanStack Query cache invalidation
```

- Configured via Vite env vars (`VITE_AOAI_ENDPOINT`, `VITE_AOAI_API_KEY`, `VITE_AOAI_DEPLOYMENT`)
- Graceful degradation: button shows toast if env vars not set
- Priority strings from AI mapped to Dataverse numeric choice keys

## Command Palette (Ctrl+K)

- Global keyboard listener (`Ctrl+K` / `Cmd+K`) mounted at app root
- Uses `cmdk` library + Radix Dialog for the modal
- Searches TanStack Query's cached data — no additional API calls
- Results grouped by entity type (Accounts, Contacts, Action Items, Meetings, Ideas)
- Fuzzy matching with highlighted text, max 5 results per group

## Key Constraints

- **Dashboard interactivity** — hover tooltips (CSS `group/tip`) + click-to-drill-down (Radix Dialog) on all dashboard tiles and sub-elements; reverse-lookup maps convert display labels to Dataverse numeric keys
- **No raw `fetch()` for Dataverse** — the iframe sandbox has no auth tokens; all Dataverse calls go through generated services. Azure OpenAI calls use `fetch()` directly (configured via env vars, not through the Power Platform host)
- **No `$select` with computed fields** — including formatted values in `$select` causes silent zero-row returns; we omit `$select` entirely
- **Dataverse only via database references** — do NOT use connector references (`shared_commondataserviceforapps`) for Dataverse
- **Generated code is read-only** — `src/generated/` and `.power/` are regenerated by `pac code add-data-source`

## Adding a New Table

```bash
pac code add-data-source -a dataverse -t <logicalname>
```

This generates `<Table>Service.ts` and `<Table>Model.ts` in `src/generated/`. Then:
1. Create TanStack Query hooks in `src/hooks/`
2. Build components in `src/components/<table>/`

## Build & Deploy

- `npm run build` → TypeScript compile + Vite production build to `dist/`
- `pac code push` → uploads `dist/` to Power Platform, registers/updates the Code App
- `power.config.json` tells `pac code push` where to find build output (`buildPath: "dist"`) and which environment to target
- **`base: "./"` in `vite.config.ts` is required** — Power Platform serves Code Apps from a nested path, not the domain root. Without this, asset paths resolve to `/assets/...` which 404s in the deployed environment
