Good news — Direct Line is not what you use for code apps. The native pattern is much cleaner. Here's what I now know with confidence:

The correct integration pattern for code apps:
You add the Microsoft Copilot Studio connector as a data source to your code app, which auto-generates MicrosoftCopilotStudioService.ts and MicrosoftCopilotStudioModel.ts. You then call ExecuteCopilotAsyncV2 to invoke Compass synchronously and get the response back directly. Microsoft Learn
The flow is:
pac code add-data-source -a "shared_microsoftcopilotstudio" -c <connectionId>
Then in your React code:
tsconst response = await MicrosoftCopilotStudioService.ExecuteCopilotAsyncV2({
  message: userInput,
  notificationUrl: "https://notificationurlplaceholder",
  agentName: "your_compass_schema_name",
  conversationId: conversationId || undefined
});
Multi-turn conversation is supported — you persist the conversationId from the response and pass it back on subsequent calls, so Compass maintains context across the conversation. Rajeevpentyala
To get the agent schema name: go to Settings → Advanced → Metadata in Copilot Studio and copy the Schema name — it typically includes a publisher prefix and is case-sensitive. Rajeevpentyala

So the architecture is clean: no Direct Line, no iframe, no web chat widget — just a typed TypeScript service generated by the SDK that you call directly from your React components. You build whatever chat UI you want around it.

That word authenticated in the URL is the tell. Because the Microsoft Copilot Studio connector is a Power Platform connector, authentication is handled entirely by the Power Apps host — the same Entra ID session the user already established when they opened the code app. You're not managing tokens, not configuring Direct Line auth, not setting up OAuth flows yourself.
For your GCC environment specifically though, you'll want to verify one thing when you go to configure Compass in Copilot Studio — under Settings → Security, make sure authentication is set to Microsoft (Entra ID) and not "No authentication." The connector integration expects the authenticated endpoint, and if the agent is set to unauthenticated it'll behave differently.
The practical upshot for your demo: a user opens the My Work code app, logs in once with their Microsoft account, and Compass just works — the connector inherits that identity automatically. No token management in your React code at all.
